## New code reviewed this pass — solid, minor notes

The onboarding, settings, and error-surfacing code is clean and idiomatic. Small notes:

- **N1 (nit):** `MurmurApp` holds `AppStatus.shared` via `@ObservedObject`; `@StateObject` is the more idiomatic ownership for the menubar-icon reactivity. Works (singleton, non-owning), but worth a manual confirm that the ⚠️ icon flips live on the first error.
- **N2 (minor):** on a paste failure with Accessibility missing, `openOnboarding()` fires every time — if the user keeps dictating without granting, the Setup window re-pops each attempt. Consider throttling to once per session.
- **N3 (judgment call, flagged by the builder):** cleanup-failure surfaces a warning every time Ollama is unreachable, even though the raw transcript still pastes. Matches the audit intent ("surface failures") but is chattier than surfacing only hard failures. Easy to downgrade to log-only.
- **N4 (cosmetic):** the DEBUG-only test-hook notification names + `EventHotKeyID` signature still read `wisprlocal`/`WSPL`. Compiled out of release, so harmless; rename for consistency whenever convenient.

## Remaining open items (from prior audit + this pass)

- **A1 (important — do before the data model changes):** SwiftData `Dictation` `@Model` has no `VersionedSchema`/migration plan. A future field change risks failing to open the store → history loss. Adopt a versioned schema now while the schema is v1.
- **A2 (minor):** `TextInjector` restores the clipboard after a fixed 1.0 s; a copy during that window is clobbered, and concurrent injections (rapid history "Insert") can race on the pasteboard.
- **A3 (minor):** if the target app quits between record-start and inject, `target.activate` no-ops and ⌘V lands in whatever's now frontmost.
- **A4 (minor):** the pill positions off `NSScreen.main` only; if the main display changes/disconnects while running, the pill isn't repositioned (could sit off-screen). Re-place on `NSApplication.didChangeScreenParametersNotification`.
- **A5 (expected):** launch-at-login registers the built app's path; only meaningful once installed to /Applications (fine — noted for distribution).
- **A6 (recommended for OSS):** no real test suite — only DEBUG standalone harnesses (`scripts/test-*.swift`). Add XCTest for the pure logic (`CleanupContext`, `TranscriptGuard`, `HistoryStore` fetch/prune, `OllamaClient` prompt assembly).
- **A7 (recommended for OSS):** no CI. A GitHub Actions workflow that runs `xcodegen generate` + `xcodebuild` (Debug + Release) on PRs would catch regressions and signal a maintained project.
- **A8 (distribution):** for a downloadable `.app` (vs build-from-source), you'd need Developer ID signing + notarization. The README's build-from-source path is the correct OSS default and needs nothing extra.
